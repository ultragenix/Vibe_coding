# CLAUDE.md — Agent_Workflow

Read `README.md` for the project vision and `AGENTS_USAGE.md` for the complete workflow.

## Rules


- **NEVER create or modify files outside of `agents/`, `skills/`, `commands/`, `templates/`, `claude-audit-guide.md`, `AGENTS_USAGE.md`, `README.md`, and `REPORTS/`.**
- All agent files are in English.
- All reports go in `REPORTS/`.
- **Skills are in `skills/`.** Each skill is a standalone file. Agents reference them in "Files to Read FIRST". Never modify a skill to be agent-specific — skills are shared.

## Git Conventions
- Commit format: `type(scope): description` (e.g. `feat(part-1): add terraform infra and docker setup`)
- Types: feat, fix, refactor, docs, chore, test
- Scope: part number or module name (e.g. part-1, ingestion, dbt, docker)
- One logical change per commit
- Never commit secrets, .env files, or large data files
- .gitignore: *.sql.gz, .env, gcp-sa-key.json, data/, terraform/*.tfstate*
- Never reference Claude, AI, LLM, or any assistant in commit messages
- Never use "Generated by", "Co-authored-by: Claude", or similar attributions
- Write commits as if a human developer wrote them
