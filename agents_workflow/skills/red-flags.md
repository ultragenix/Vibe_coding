# Skill: Red Flags — Universal Anti-Rationalizations

These warning signs indicate you are deviating from the process.
If you think any of these, STOP and reconsider.

## Dangerous shortcuts
- "Too simple to follow the process" → The process IS simple. Follow it.
- "Just this once, I'll skip this step" → No. Exceptions become habits.
- "I'll fix it later" → "Later" never comes.
- "It works on my machine" → Prove it works in the project context.

## Role confusion
- "While I'm at it, I'll also..." → STOP. Stay in your role. Redirect to the correct agent.
- "It's a small fix, no need for QA" → Yes there is. Every change goes through the full cycle.
- "I'll just reorganize this code while I'm here" → That's the tech-lead-dev's job, not yours.

## Communication
- "The user will understand" → No. Announce before acting. Explain what you're doing.
- "I'll do everything at once and present the result" → Announce first, confirm, then act.

## Quality
- "The code compiles, so it's fine" → Compilation != correctness.
- "Tests pass" → Did you RUN them in THIS session? Fresh evidence only.
- "Same as before" → If it's the same, why change it? If it's different, test it.

## Escalation
- "One more attempt" (3rd+) → STOP. This is an architecture problem.
- "I don't understand but I'll continue" → STOP. Say what you don't understand.
- "I'll work around the problem" → STOP. Solve the root cause.
